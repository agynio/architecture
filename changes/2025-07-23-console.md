## Delta

The Console application does not exist yet. The following are not implemented:

- Console SPA (`agynio/console-app` repository)
- Console Kubernetes deployment and Istio VirtualService routing (`console.agyn.dev`)
- Gateway: `OrganizationsGateway`, `LLMGateway` proto services
- Gateway: extended `UsersGateway` (CreateUser, GetUser, ListUsers, UpdateUser, DeleteUser)
- Gateway: extended `RunnersGateway` (RegisterRunner, GetRunner, ListRunners, UpdateRunner, DeleteRunner, ListWorkloads, GetComputeUsage)
- Gateway: extended `SecretsGateway` (full CRUD for secret providers and secrets)
- Gateway: extended `TokenCountingGateway` (GetUsageSummary)
- Users service: `CreateUser`, `GetUser`, `ListUsers`, `UpdateUser`, `DeleteUser` methods (admin CRUD)
- Users service: rename internal `CreateUser` to `ProvisionUser`
- Runners service: `ListWorkloads`, `GetComputeUsage` methods
- Token Counting service: `GetUsageSummary` method
- Bootstrap Terraform: create real admin user via Gateway API using synthetic admin token
- Terraform provider: `agyn_user` resource

## Acceptance Signal

- Console SPA deployed at `console.agyn.dev`, accessible via OIDC login.
- Cluster admin can manage users, cluster-scoped runners, and cluster-scoped apps.
- Organization owner can manage org resources (agents, providers, models, secrets, runners, members).
- Monitoring section displays active workloads, storage, and usage metrics.
- Bootstrap flow provisions a real OIDC-backed admin user via the platform API.
