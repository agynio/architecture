# Usage

## Purpose

Usage gives org admins realtime visibility into platform resource consumption. It shows how much LLM capacity, compute, storage, and platform resources are consumed across agents and identities.

## User Stories

- As an org admin, I want to see total LLM token usage across my organization so I can monitor spend.
- As an org admin, I want to see LLM usage broken down by consumer and by model so I can identify heavy consumers and expensive models.
- As an org admin, I want to distinguish input, cached, and output tokens so I can assess caching efficiency.
- As an org admin, I want to see request success and failure counts so I can detect reliability issues.
- As an org admin, I want to see compute and storage usage per agent so I can understand infrastructure consumption.
- As an org admin, I want to filter by time range so I can compare usage across periods.
- As an org admin, I want data to reflect the current state when I load the page.

## Behavior

### Time Range

A time range selector controls the window for all metrics on the page. Changing the range reloads all metrics.

Supported ranges:

- Last 24 hours
- Last 7 days
- Last 30 days
- Custom date range

### LLM

Tracks token consumption and request outcomes for all LLM calls.

**Metrics:**

| Metric | Description |
|--------|-------------|
| Input tokens | Tokens in prompts sent to models |
| Cached tokens | Input tokens served from the provider's prompt cache |
| Output tokens | Tokens generated by models |
| Requests | Total LLM calls, split by success and failed |

**Charts:**

- **Daily usage** — bar chart showing input, cached, and output tokens per day over the selected time range. Stacked bars distinguish token types.
- **By consumer** — bar chart showing total tokens (input + output) per identity, sorted descending. Allows identifying the heaviest consumers.
- **By model** — bar chart showing total tokens per model, sorted descending. Allows comparing cost distribution across models.

### Compute

Tracks CPU and RAM allocated to running agent workloads.

**Metrics:**

| Metric | Description |
|--------|-------------|
| CPU-core-seconds | Allocated CPU × seconds across all workloads |
| RAM-GB-seconds | Allocated RAM × seconds across all workloads |

**Charts:**

- **Daily usage** — bar chart showing CPU-core-seconds and RAM-GB-seconds per day over the selected time range.
- **By consumer** — bar chart showing total compute (CPU and RAM separately) per agent identity, sorted descending.

### Storage

Tracks persistent volume capacity allocated to agents.

**Metrics:**

| Metric | Description |
|--------|-------------|
| Storage-GB-seconds | Allocated volume size × seconds across all persistent volumes |

**Charts:**

- **Daily usage** — bar chart showing storage-GB-seconds per day over the selected time range.
- **By consumer** — bar chart showing total storage per agent identity, sorted descending.

### Platform

Tracks creation of platform entities.

**Metrics:**

| Metric | Description |
|--------|-------------|
| Threads created | Threads created in the selected period |
| Messages sent | Messages sent in the selected period |

**Charts:**

- **Daily usage** — bar chart showing thread and message counts per day over the selected time range.

## Constraints

- Compute and storage usage is based on allocated resources, not measured consumption.
- Data reflects the state at the moment the page loads. Reload to see changes.
